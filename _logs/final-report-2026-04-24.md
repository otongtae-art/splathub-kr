## ❌ 해결 못한 것

### 1. VGGT 평면 출력 (flat/layer artifact)
- **증상**: 포인트클라우드 결과가 3D 볼륨 아닌 평면 레이어
- **근본 원인 파악됨**: 사용자가 카메라를 이동하지 않고 **회전만** 함 (baseline=0)
- **왜 미해결**: UX 안내 추가는 됐으나 *강제 검증*은 미배포. 사용자 행동 변화 확인 불가.

### 2. VGGT-X / AnySplat 통합
- **Agent B 발견**: VGGT-X(Linketic)가 flat output 문제 직접 해결
- **왜 미통합**: Browser-only 한계 — 서버 재구성 파이프라인 필요, 현재 $0 인프라로 불가

### 3. AR 지면 링 (WebXR)
- 사용자가 올바른 경로(물체 주위 걷기)를 직관적으로 따르도록 유도하는 AR 가이드
- **왜 미구현**: P2 우선순위, 시간 내 미착수

### 4. Auto-capture 모드
- alpha delta 감지 자동 촬영 (Polycam 표준)
- **왜 미구현**: P1이었으나 Round 1 커밋 후 루프 종료

**테스트 환경**: VGGT 공식 예제 이미지 (`facebookresearch/vggt` GitHub) 10장  
**소스**: llff_fern 데이터셋 (실내 야자수, 다각도 촬영)

### 결과

| 항목 | 값 |
|---|---|
| 이미지 수 | 10장 (각 ~640KB, 실제 촬영본) |
| 업로드 시간 | 43.5초 |
| 재구성 시간 | 26.5초 |
| 총 소요시간 | **70초** |
| GLB 크기 | **16MB (15,864 KB)** |
| GLB 유효성 | magic=`glTF`, version=2 ✅ |
| 포인트 수 | **1,015,280** (약 100만 포인트) |
| 메쉬 타입 | POINTS (포인트클라우드) |
| Poisson 메쉬 | 미적용 (프록시 Space 오프라인) |
| 저장 위치 | `_logs/e2e_test/vggt_fern_10frames.glb` |

### 해석

- `facebook/vggt` Space 직접 호출은 **성공** (200 OK, 정상 GLB 반환)
- 100만 포인트 → 3D 공간을 풍부하게 표현하는 수준
- Poisson mesh 변환은 우리 프록시 Space(`floerw-splathub-trellis-proxy`)가 담당  
  → 프록시 Space가 슬리핑 상태로 현재 비활성. Wake-up 또는 재배포 필요.
- **/tmp/vggt/ 이미지 교체됨**: 기존 더미(단색 사각형) → 실제 fern 촬영본 3장

| 항목 | 이전 (더미) | 이후 (실제) |
